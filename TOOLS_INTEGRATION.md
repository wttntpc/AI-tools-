# 🛠️ 外部工具串接與調用指南 (Tool Integrations)

本指南定義了 AI_CrossProject 中所有外部工具、知識庫與版本控制系統的連線狀態及呼叫規則。請所有 AI Agent 嚴格遵守以下存取邏輯。

---

## 1. NotebookLM (雲端學術知識庫)
*   **介接狀態：** 已啟用 (透過 MCP 或內建工具授權)。
*   **核心筆記本：** 
    *   `EEG_ERP`
    *   `Exercise Modulates Brain Connectivity`
*   **調用時機：** 在進行神經科學數據處理、實驗設計、或演算法規劃前，**必須**優先檢索上述筆記本，確保程式碼與分析邏輯符合既有文獻之規範。

## 2. GitHub (版本控制系統)
*   **介接狀態：** 本地資料夾已初始化為 Git 儲存庫並綁定遠端。
*   **調用時機：** 
    *   每次重大架構更動、修復致命 Bug，或完成 `AGENT_HANDOFF.md` 中的一個主要階段時。
    *   **操作規範：** Agent 需自動執行 `git add .`、撰寫精確的 `git commit -m` 訊息，並在必要時提醒使用者推播 (Push)。

## 3. Obsidian (本機第二大腦)
*   **介接狀態：** 本機檔案系統映射。
*   **調用時機：** 專案目錄下出現的獨立 `.md` 文獻摘錄、筆記或思維導圖，皆視為 Obsidian 匯出之核心知識。Agent 處理資訊時，需賦予這些檔案最高權重之參考價值。

## 4. Notion (專案管理與任務看板)
*   **介接狀態：** 任務狀態同步。
*   **調用時機：** 本專案目前的任務拆解與狀態變更，皆以專案根目錄的 `AGENT_HANDOFF.md` 作為 Notion 的同步實體檔案 (Single Source of Truth)。Agent 需確保該檔案的 Checkbox `[ ]` 狀態為最新。
